What is Multimodal AI?

AI systems that work with and generate multiple types of content beyond text — including images, video, code, and structured data. In business, combining modalities enables professionals to create deliverables that previously required specialised expertise in design, videography, programming, or data science.

🖼️
Visual AI
Professional images from text descriptions. No Photoshop, no photographer.

Tools: Gemini (Nano Banana), ChatGPT image gen, Reve Image, Adobe Firefly
🎬
Video AI
Dynamic video content & AI presenters from scripts. No production crew needed.

Tools: HeyGen (Avatar III — free), Kling, Runway
💻
Code AI
Functional automations from plain English. No programming degree required.

Tools: GitHub Copilot, ChatGPT, Claude
📊
Synthetic Data
Realistic datasets without privacy risk. Perfect for prototyping & dashboards.

Tools: Mockaroo, ChatGPT, Claude

This Landscape Changed — Just This Week

Context

The tools on the previous slide weren't all available last semester. Several major changes landed in the last 8–12 weeks alone:

🖼️ Image
DALL·E 3 retired — GPT Image 1 launched
OpenAI's image engine changed entirely in late 2025. ChatGPT now uses GPT Image 1, which is significantly better at following complex instructions and rendering text accurately. "DALL·E 3" is already outdated terminology.

Reve Image also emerged from nowhere in early 2025 and immediately topped global benchmarks for prompt accuracy.
🎬 Video
HeyGen Avatar IV — uncanny valley largely solved
HeyGen's November 2025 update to Avatar IV eliminated most of the artificial, stilted look of earlier AI presenters. The same tool you'd have used 6 months ago now produces noticeably more natural results — though Avatar IV costs premium credits on the free plan.
💻 Code
Copilot Free tier launched — no student account needed
GitHub introduced a permanently free tier of Copilot in late 2025 — 2,000 code completions per month with no credit card or student verification required. This removes the previous barrier for students whose Student Pack application hadn't been approved yet.
⚡ Velocity
Expect the same again before Week 10
The tools on this slide will likely be superseded or significantly updated before this course ends. This is the skill: staying current, evaluating new tools critically against existing ones, and knowing when to switch. The adaptability matters more than any single tool.

RCTF Still Works — Across Every Modality

Framework

The prompting framework you mastered in Week 1 transfers directly. The output type changes; the thinking does not.

R
Role
Image: "expert product photographer"
Code: "senior Python developer"
Video: "professional scriptwriter"
Data: "business analyst"
C
Context
Who the audience is, what platform the output will appear on, and what business problem you're solving
T
Task
The specific deliverable — "generate a product hero shot", "write an Excel macro", "create a 30-second welcome video script"
F
Format
Image: "16:9, white background, soft lighting"
Code: "Python, with comments, no dependencies"
Data: "CSV, 50 rows, realistic Australian names"

Key insight: You're not learning four new skills — you're applying one core skill (effective AI collaboration) to four different output types.

Divide & Conquer

Group Roles

Allocate roles within your group based on interest — everyone documents their own contributions in their AI Prompt Log for Assignment 3.

Suggested Division
  • Visual Lead — drives image generation, manages prompt iterations, selects final outputs
  • Video Lead — writes the video script using RCTF, manages HeyGen export

Is It Actually Good Enough?

Evaluation Framework

For each AI output you produce today, apply this four-question test before calling it done:

🖼️ Images — Check For
· Distorted or inconsistent features (hands, text, logos)
· Lighting inconsistencies or unrealistic shadows
· Brand appropriateness for your specific audience
· Whether it communicates the intended message
🎬 Video — Check For
· Uncanny valley effect in avatar expressions
· Lip sync accuracy with the script
· Whether the script sounds natural when spoken
· Watermarks on free-tier exports
💻 Code — Check For
· Does it actually run? (Test it — don't assume)
· Does the output match expected results?
· Are there security or privacy risks?
· Could you explain the logic to a colleague?
📊 Data — Check For
· Is the distribution realistic? (Not all customers aged 25)
· Are Australian formats used? (Postcodes, state codes, phone numbers)
· Always disclose: "This is synthetic data for illustration only"
· Never use for actual business decisions
